# 04 — Distribution

## 分发策略

分享结束后，按优先级分批分发。SwenChan（主力）→ 私域（补充）→ 资产沉淀（长尾）。

---

### 渠道 1：Twitter/X — @swen_chan

**优先级：** 🔴 高（主力阵地，目标 2026 年底 1000-2000 粉）

**内容形式：**
- 主推文：用一句话 hook（"我用 Hermes 准备了一场关于 Hermes 的分享"）+ 2-3 张 slides 精选截图
- 可选第二推：放 Demo 视频片段（压缩版 30-60s）+ 搭建指南链接
- 可选第三推：分享过程中的意外/坑/心得（增加真实感）

**时机：** 分享当天或第二天（趁热）

**注意事项：**
- Hook 要锋利，不要写"今天参加了 OPC 大理创客松"——直接上 meta-demo 概念
- Slides 截图建议选 Slide 5（Live Demo + Meta 逻辑）和 Slide 3（Hermes 是什么）
- 推文最后引导关注："更多 Agent 搭建/商业化思考，关注获取"
- 考虑 thread 形式（如果内容多）而非单条长推

---

### 渠道 2：公众号 — Swen View

**优先级：** 🟡 中（私域被动分发，~3000 粉）

**内容形式：**
- 长文：完整分享内容 + 搭建指南 + meta-demo 过程复盘
- 标题方向：偏向实用教程而非活动报道——"我用 AI Agent 准备了一场关于 AI Agent 的分享，过程全记录"
- 文末附：搭建指南 PDF 获取方式 / Twitter 关注引导

**时机：** 分享后 1-3 天内（精细打磨）

**注意事项：**
- Swen View 公众号目前仅保留每年一篇年度经济复盘（2020-2026），这篇属于例外内容，注意不要打乱既有节奏
- 如需发，可以作为"公众号特别篇"发布，而非常规更新
- 去掉 Web3 相关内容——中文公域需注意政策风险

---

### 渠道 3：即刻

**优先级：** 🟡 中

**内容形式：**
- 短帖 + 精选 slides 截图（2-3 张）
- 文案风格与 Twitter 类似但可更轻松，即刻氛围偏技术社群

**时机：** 与 Twitter 同步或稍晚

**注意事项：**
- 尚未注册——如果要发，提前注册账号
- 即刻对 AI Agent 话题接受度较高，可以适当加入细节

---

### 渠道 4：LinkedIn

**优先级：** 🟢 低（~400 connections）

**内容形式：**
- 英文摘要版（3-5 段）：What is Hermes, the meta-demo concept, key takeaways
- 附 slides 截图 + Demo 视频链接

**时机：** 分享后 2-3 天

**注意事项：**
- 英文写作注意文化和语境自然，不突兀（参见 user profile 风格要求）
- LinkedIn 适合建立专业形象，但非当前主攻方向

---

### 渠道 5：GitHub

**优先级：** 🟡 中（portfolio 资产）

**内容形式：**
- 整个 `Hermes-Demo` 目录上传为公开 repo：`swen-chan/hermes-demo`
- README.md 已就绪
- 可加 GitHub Pages 展示 slides（可选）

**时机：** 分享后 1 周内

**注意事项：**
- 检查所有文件中无敏感信息（API key 路径、个人隐私）再 push
- 确认 `03-hermes-guide.md` 中的示例 SOUL/AGENTS 不包含你的真实配置
- Repo 作为长期 portfolio 资产——以后分享类似主题直接引用

---

### 渠道 6：小红书 / 知乎 / 知识星球（可选）

**优先级：** ⚪ 低（Swen View 既有渠道，~3000 / ~1000 / ~10 人）

**内容形式：** 从公众号长文精简或截图版

**时机：** 视精力决定，可跳过

**注意事项：**
- 小红书偏生活化——需要重新包装视觉和文案
- 知乎适合图文教程形式，可以直接贴搭建指南
- 知识星球不做主动增长，仅视情况同步
