## Slide 3 — 从 OpenClaw 到 Hermes

**标题：** 从 OpenClaw 到 Hermes

**顶部 hook：**
> OpenClaw 打开了想象力，结合社区最佳实践进化成了 Hermes。

**三卡片（设计选择，非功能对比）：**

| 卡片 | 内容 |
|---|---|
| 🧠 不只是对话 | SOUL 定义人格，AGENTS 约束边界，Skills 积累经验。一次配置，持续生效。 |
| 💾 不从头开始 | Memory 跨 session 持久化。下次打开不用重新交代你是谁、怎么干活。 |
| 🛡️ 可控才能持续 | 危险操作等你确认。信任不是说说——是设计出来的。 |

**底部收束：**
> 从想象力到日常。从单次惊艳到持续可用。

**备注（口播）：**
- OpenClaw 和 Hermes 本质上是同类工具——都有 SOUL、Memory、Skills、多平台接入
- 区别不在 feature，在哲学：Hermes 是 OpenClaw 社区实践沉淀后的进化版
- 三个关键设计选择：工作系统化（不是一次 prompt）、持久记忆（不从头聊）、可控执行（信任才能持续用）
- 核心理念：你定原则，它干活；你审结果，它改进
